- Stack: Static HTML generated by PowerShell scripts

## Codex Scope: articles.json

Codex is responsible **only** for adding and editing entries in `articles.json`.

### Article Schema

```json
{
  "id": "unique-slug-in-kebab-case",
  "title": "Article Title in English (60 chars max)",
  "excerpt": "One or two sentences shown on cards and in meta description. 120 chars max.",
  "body": [
    "Paragraph or section content as individual strings.",
    "Each string renders as a separate paragraph.",
    "Minimum 3 paragraphs, maximum 8."
  ],
  "category": "travel-guide",
  "tags": ["kyoto", "first-time"],
  "publishedAt": "2026-05-20",
  "updatedAt": "",
  "readingTime": 6,
  "heroImage": "https://example.com/image.jpg",
  "heroImageAlt": "Descriptive alt text for the hero image",
  "heroImageCredit": "Source Name",
  "affiliate": false,
  "affiliateLinks": [],
  "relatedIds": [],
  "author": "TABI Editorial"
}
```

### Field Rules

| Field | Rule |
|-------|------|
| `id` | Unique, kebab-case, URL-safe, no Japanese |
| `title` | English, 60 chars max, no clickbait |
| `excerpt` | 1-2 sentences, 120 chars max, no "click here" |
| `body` | Array of strings (paragraphs), min 3 max 8 |
| `category` | Must be one of the 5 defined values (see below) |
| `tags` | 2-5 items from defined taxonomy (see below) |
| `publishedAt` | `YYYY-MM-DD` format |
| `updatedAt` | `YYYY-MM-DD` or empty string `""` |
| `readingTime` | Integer minutes, calculated from body length |
| `heroImage` | Full URL to image. Use article OG image when available |
| `heroImageAlt` | Required when heroImage is set |
| `heroImageCredit` | Source publication or photographer name |
| `affiliate` | `true` only for `things-to-buy` category |
| `affiliateLinks` | Array of `{"label": "...", "url": "...", "price": "..."}` |
| `relatedIds` | Array of existing article `id` values |
| `author` | Default: `"TABI Editorial"` |

### Category Values (use exactly as written)

| Value | Display Label |
|-------|--------------|
| `travel-guide` | Travel Guide |
| `culture` | Culture & Tradition |
| `food` | Food & Drink |
| `things-to-buy` | Things to Buy |
| `hidden-gems` | Hidden Gems |

### Tag Taxonomy (use only these tags)

`first-time` / `tokyo` / `kyoto` / `osaka` / `nature` / `food` /
`tradition` / `festival` / `shopping` / `budget` / `luxury` /
`off-the-beaten-path` / `transport` / `accommodation` / `affiliate`

## Content Guidelines

**Tone**: Warm, knowledgeable, insider — not a tourist pamphlet, not a Wikipedia article.

### Title
- Specific and benefit-driven: *"How to Climb Mt. Fuji Without Hiring a Guide"*
- Avoid vague: ~~"Things to Do in Kyoto"~~
- No emoji, no ALL CAPS

### Excerpt
- Lead with the most interesting fact or hook
- Never start with "In this article..." or "Learn about..."
- Should make the reader want to continue

### Body paragraphs
- Each paragraph: one idea, 2-4 sentences
- Use concrete details: place names, prices in ¥, train lines, hours
- First paragraph: expand on excerpt hook
- Last paragraph: practical takeaway or call to action

### Things to Buy articles
- Always include actual prices (¥ and approx USD)
- Explain *why* it's worth buying — not just what it is
- `affiliateLinks` required for each recommended product

## What NOT to Do

- Do not edit `site.config.json`, `styles.css`, `script.js`, or any `.ps1` scripts
- Do not create or delete HTML files (they are generated)
- Do not modify `AGENTS.md` or `CLAUDE.md`
- Do not add Japanese text to any field (English-only site)
- Do not invent tags outside the defined taxonomy
- Do not set `affiliate: true` for non `things-to-buy` articles
